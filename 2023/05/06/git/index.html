# 一、学习 git 的意义

# 1、在协同工作可能会出现的问题

我们在工作的时候肯需要多人协作完成同一个项目，在这个时候可能会出现以下问题

  • 文件版本混乱
  • 无法同时完成分别的任务
  • 项目一旦出问题，回退复杂

这个时候版本控制软件就很有用了！！！

# 2、版本控制软件的类别

  • 集中式版本控制软件

​ 缺点：

​ (1) 一旦服务器出现问题大崩盘，所有成员都无法接收或提交文件

​ (2) 多人协作会出现版本混乱的问题

​ (3) svn 虽然后期利用锁定解决了多个用户修改文件出现的版本问题，但是不能同时工作，每次只能有一人在线修改

​ (4) 用户每次都需要将文件上传到中央服务器，一旦网络比较差 / 文件很大，耗费时间多，对网络依赖性强

  • 分布式版本控制

​ 特点：每一名开发者的电脑上都会有一个完整的版本库，这样你就不用在开发工作时不需要联网（因为版本库就在个人开发者的电脑上）, 那么既然每个人都有完整的版本库。

​ 工作流程：

​ (1) 开发者先将远程服务器的完整的版本库拷贝到本地

​ (2) 开发者在本地修改对应的功能模块 - (本地仓库会自动生成版本文件信息)

​ (3) 如果有人修改文件，可以更新文件

​ (4) 提交前查看修改内容

​ (5) 将本地库的对应版本文件推送到远程服务器中 - (服务器此时会比对本地库的文件和服务器文件区别，在生成版本信息)

​ (6) 若发现错误，撤回推送，重新提交

​ 相比集中式版本控制的优点：

​ (1) 安全性高 - (一旦一个人电脑坏了，还可以去另一个人那里复制，文件不会丢失)

​ (2) 减少了版本冲突，虽然也会有版本冲突的问题，但是减少了版本冲突，不会用户每一个修改文件都会引起大冲突，在将最终文件推送到服务器时需要解决

​ (3) 对网络依赖性减弱，即使服务器出现问题，用户还可以在本地仓库中修改文件

总结，版本控制就是用于管理多人协同开发项目的技术。

# 3、git 是啥？

​ git 是一个免费开源、高效快速的分布式版本控制软件

# 4、git 的特点

  • 并非基于差异对比变更内容，而是直接记录版本信息
  • 几乎所有操作都是基于本地执行
  • git 能保证文件传输的完整性 - 底层：Git 中所有的数据在存储前都计算校验和，然后以校验和来引用。
  • 有极强的分支管理

# 二、Git 工作区、暂存区和版本库

  • ** 工作区：** 就是你在电脑里能看到的目录。
  • ** 暂存区：** 英文叫 stage, 或 index。一般存放在 “.git 目录下” 下的 index 文件（.git/index）中，所以我们把暂存区有时也叫作索引（index）。
  • ** 版本库：** 工作区有一个隐藏目录.git，这个不算工作区，而是 Git 的版本库。
